# 아이폰 단축어와 AI로 만드는 '실시간 금융 장부' 무인 자동화 가이드

## 1. 개요

### 목적
매일 쏟아지는 은행과 카드사의 결제 문자 메시지, 일일이 장부에 옮겨 적기 귀찮으셨죠? 이 가이드의 핵심은 **OpenClaw(AI 에이전트)**가 보스를 대신해 **Notion**에 회계 장부를 기록하고 통계까지 산출하는 **완전 무인 자동화 시스템**을 구축하는 것입니다. 사업체의 회계 관리를 AI에게 맡기고 보스는 의사결정에만 집중하세요.

### 시스템 프로세스 (Workflow)
전체적인 흐름은 다음과 같습니다:

`문자메시지 수신` -> `아이폰 단축어 처리` -> `Notion 실시간 거래 내역 DB` -> `OpenClaw 데이터 해석` -> `Notion Cashflow Dashboard` -> `월간 예산/계획 DB 반영`

---

## 2. 단계별 구축 가이드

### STEP 1: 아이폰 단축어 설정 (데이터의 관문)
아이폰의 '자동화' 기능을 이용해 특정 발신번호로 문자가 오면 즉시 노션으로 텍스트를 전송하도록 구성합니다. 이 단계에서 가장 중요한 것은 데이터의 '전처리'입니다.

![단축어 자동화 설정 1](/assets/images/shortcut_step1_1.jpg)
*문자 수신 시 자동으로 실행될 단축어를 생성합니다.*

*   **핵심 팁 (텍스트 대치)**: 은행 문자는 가독성을 위해 줄바꿈(Enter)이 많습니다. 하지만 이 줄바꿈이 포함된 채로 JSON 데이터를 전송하면 API 에러가 발생하여 동작하지 않습니다. 
*   **해결책**: 단축어 설정에서 **'텍스트 대치'** 기능을 사용해 모든 **"줄바꿈"**을 **"공백(스페이스)"**으로 치환해야 합니다. 화면상으로는 단순히 공백으로만 보이지만, 이 한 끝 차이의 디버깅이 시스템의 생존을 결정합니다!

![단축어 자동화 설정 2](/assets/images/shortcut_step1_2.jpg)
*텍스트 대치와 JSON 본문 구성 예시. 줄바꿈을 공백으로 바꾸는 것이 핵심입니다.*

![단축어 자동화 설정 3](/assets/images/shortcut_step1_3.jpg)
*Notion API 헤더 설정 및 POST 요청 구성 화면입니다.*

### STEP 2: Notion 실시간 거래 내역 DB 업데이트
단축어가 보낸 원문 데이터가 가장 먼저 도착하는 곳입니다. 여기에는 정제되지 않은 문자 메시지 전문이 `제목` 필드에 쌓이게 됩니다.

![실시간 거래 내역 DB](/assets/images/notion_raw_transactions.jpg)
*가공되지 않은 원문 데이터가 Processed 체크박스와 함께 저장됩니다.*

### STEP 3: OpenClaw 데이터 분석 (지능형 엔진)
이제 AI 에이전트 OpenClaw(JARVIS)가 나설 차례입니다. 서버의 `cron` 기능을 이용해 **매 30분마다** 자동으로 깨어나 처리되지 않은 데이터를 분석합니다. 

단순히 글자를 옮기는 것이 아니라, 문맥을 파악하여 **1,000원 미만의 소액**까지 정확히 잡아내는 지능적인 파싱을 수행합니다.

```javascript
// scripts/hcc_processor.js 핵심 파싱 로직
const title = page.properties['제목'].title[0].plain_text;

// 2. Parse data from text using Regex
const typeMatch = title.match(/(입금|출금)/);
const amountMatch = title.match(/금액([\d,]+)/);
const balanceMatch = title.match(/잔액([-?\d,]+)/);
const counterpartyMatch = title.match(/(입금|출금)\s+(.*?)\s+금액/);
const timeMatch = title.match(/(\d{2}\.\d{2}\s\d{2}:\d{2})/);

const type = typeMatch ? typeMatch[1] : 'Unknown';
const amount = amountMatch ? parseInt(amountMatch[1].replace(/,/g, '')) : 0;
const balance = balanceMatch ? parseInt(balanceMatch[1].replace(/,/g, '')) : 0;
let counterparty = counterpartyMatch ? counterpartyMatch[2].trim() : 'Unknown';
const timestamp = timeMatch ? `2026-${timeMatch[1]}` : new Date().toLocaleString();
```

### STEP 4: Notion Cashflow Dashboard 업데이트
분석이 끝난 데이터는 깔끔하게 정리되어 'Cashflow Dashboard' DB로 옮겨집니다.

![Cashflow Dashboard](/assets/images/notion_cashflow_dashboard.jpg)
*날짜, 금액, 카테고리, 그리고 실시간 잔액이 정교하게 기록됩니다.*

### STEP 5: Notion 월간 예산/계획 DB 반영
마지막 단계는 통계입니다. 거래 내역이 기록됨과 동시에 해당 월의 카테고리별 예산 항목과 자동으로 연결됩니다.

*   **자동화의 묘미**: 매달 반복되는 예산 항목 입력이나 과거 데이터 정비 같은 번거로운 작업은 OpenClaw에게 **"일괄 작업"**을 시켜 한 번에 끝낼 수 있습니다. 보스는 그저 그래프로 시각화된 집행률만 확인하면 됩니다.

![월간 예산 및 실적 현황](/assets/images/notion_budget_plan.jpg)
*항목별 월간 예산 대비 실적이 실시간 집행률(🟩🟩🟩⬜⬜)로 표시됩니다.*
