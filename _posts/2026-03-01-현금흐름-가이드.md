# 아이폰 단축어와 AI로 만드는 '실시간 금융 장부' 무인 자동화 가이드

## 1. 개요
"내 폰 안의 경리, AI 에이전트" – 단순히 입금 문자를 받는 것에 그치지 않고, AI가 문자를 해석해 장부를 만들고 대시보드까지 자동 업데이트한다면 어떨까요? 아이폰 단축어와 **OpenClaw (AI 에이전트)** 를 결합해, 손 하나 까딱하지 않고 구축하는 실시간 금융 자동화 시스템을 소개합니다.

## 2. 단계별 구축 가이드

### STEP 1: 아이폰 단축어 – 문자를 Notion 으로 전송
가장 먼저 아이폰이 받은 문자를 Notion DB 로 전달해야 합니다. (이미지 1, 2, 3 참조)

- **핵심 설정**: ‘메시지를 받을 때’ 자동화를 만들고, 수신된 메시지 전문을 Notion API 로 전송하도록 구성합니다.

![단축어 설정 이미지 1](/assets/images/step1_img1.jpg)
![단축어 설정 이미지 2](/assets/images/step1_img2.jpg)
![단축어 설정 이미지 3](/assets/images/step1_img3.jpg)

- **필수 팁 (텍스트 대치)**: 은행 문자는 줄바꿈(엔터)이 많아 JSON 전송 시 에러의 주범이 됩니다. 단축어의 **‘텍스트 대치’** 기능을 사용해 줄바꿈을 공백으로 치환해주는 전처리를 반드시 거쳐야 합니다. (이미지 1, 3 참조)

### STEP 2: Notion 데이터베이스 준비
문자가 쌓일 그릇을 만들어요. (이미지 4 참조)

- 단축어가 보낸 원문 데이터가 **Title** 필드에 차곡차곡 쌓이게 됩니다.
- 에이전트가 처리했는지 구분할 수 있도록 **Processed** 체크박스 속성을 추가해 주세요.

![Notion DB 저장 이미지](/assets/images/step2_img.jpg)

### STEP 3: AI 에이전트 데이터 분석 (OpenClaw)
Notion에 쌓인 원본 문자를 AI가 읽고 분석합니다.

- **추출 로직**: AI는 문맥 속에서 `[입금/출금]`, `금액`, `거래처`, `잔액` 을 정확히 뽑아냅니다. 1,000원 미만의 소액(예: 109원)도 누락 없이 잡아냅니다.

### STEP 4: 실시간 대시보드 업데이트
분석된 데이터는 즉시 **‘제국 병원 상황실’** 대시보드에 반영됩니다. (이미지 5 참조)

- **시각화**: 현재 가용 시제를 콜아웃 블록에 표시하고, 하단에 최근 거래 내역 5건을 불렛 포인트로 나열해 한눈에 들어오게 구성했습니다.

![분석·새 DB 저장 이미지](/assets/images/step3_img.jpg)

## 3. 구축 시 핵심 주의사항 (Troubleshooting)
- **JSON 데이터 깨짐 방지**: 앞서 강조했듯 문자의 줄바꿈은 API 전송 실패의 가장 큰 원인입니다. 이미지 3처럼 단축어 설정에서 반드시 **텍스트 정제(대치)** 과정을 포함해야 시스템이 멈추지 않고 돌아갑니다.
